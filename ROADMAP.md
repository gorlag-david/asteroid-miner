# lazygamedev Roadmap

## Mission

Ship a playable game. Start simple, iterate fast, ship something people can actually play.

## Phase 1: Foundation (Current)

Get the technical foundation in place so we can build on solid ground.

- **Project scaffold**: Initialize repo, set up build tooling, pick a game framework
- **Game concept**: Define what we're building — genre, core mechanic, scope
- **Basic game loop**: Render a window, handle input, update/draw cycle working

## Phase 2: Prototype

Build the minimum viable game loop that proves the core mechanic is fun.

- **Core mechanic implementation**: The one thing the player does most
- **Basic art/assets**: Placeholder or simple assets — enough to playtest
- **Game state management**: Start, play, pause, game over
- **Basic UI**: Title screen, HUD, game over screen

## Phase 3: First Playable

Polish the prototype into something worth sharing.

- **Content**: Enough levels/variation to keep a player engaged for 5+ minutes
- **Audio**: Sound effects and background music
- **Polish**: Juice, feedback, screen shake — make it feel good
- **Build & distribution**: Packaged build someone can download and run

## Principles

- **Ship over perfect.** A finished simple game beats an unfinished ambitious one.
- **Playtest early.** If the core loop isn't fun with placeholder art, more art won't fix it.
- **Small scope.** One mechanic, done well.
- **Web-first.** Browser games have zero install friction. Default to web unless there's a strong reason not to.
